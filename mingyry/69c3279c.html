Git

Git 常用命令

git init

初始本地库

默认分支名字 master

git status

查看本地库状态

git add

添加文件到暂存区

git commit -m “ ” 文件名

提交文件， 并且添加注释

git reflog

查看版本日志信息

git log

查看详细版本信息

git reset —hard 版本号

穿越版本

git分支模式

git分支操作

  • git branch 分支名

    创建分支

  • git branch -v

    查看分支

  • git checkout 分支名

    转换到当前分支名

  • git merge 分支名1

    合并分支名1到当前的分支

git冲突

git无法自动合并两个文件，需要人为解决冲突

  1. 解决冲突

  2. 添加到暂存区

  3. 提交文件

    不能带文件名的提交

Github

创建远程库

查看远程库别名

git remote -v

创建远程库别名

git remote add 别名 原名

推送远程库

git push 别名 分支

推送远程库的最小单位是分支

拉取远程库

git pull 别名 分支

克隆远程库

git clone 地址

不需要登录账号

clone：拉取代码，初始化本地库，创建别名

团队协作

  1. 修改文件，推送到远程库

  2. 需要主创建人邀请合作者

  3. 合作者复制邀请函，接受邀请，才可以推送到远程库

跨团队协作

  1. 点击fork 插入到自己的仓库中

  1. pull request 请求提交到对方的库中

  1. 主创建人，接收提交

SSH免密码登录

  1. 生成ssh密钥

    ssh-keygen -t rsa -C 用户邮箱

  1. 复制公钥 .pub文件是公钥
  2. 添加公钥到github上
  3. 利用SSH拉取代码

IDEA集成Github

  1. 设置github账号

  2. 也可以使用Token登录

  3. 分享项目到github

  1. 推送分支到远程库，可以调整推送到哪个库，利用remote进行设置

    如果本地库代码跟远程库代码版本不一致，推送操作会被拒绝，因此，想要push成功，一定要保证本地库的版本要比远程库的版本高

    先检查远程库和本地代码的区别，如果本地代码版本落后，切记先pull远程库的代码

  2. 拉取远程库的代码

  3. clone 克隆远程库代码到本地
